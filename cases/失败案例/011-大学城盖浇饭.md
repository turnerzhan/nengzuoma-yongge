# 案例 011：大学城盖浇饭

> 一句话摘要：39 万开在新建大学城，幻想日卖 8000。

## 基本信息

| 项 | 内容 |
|---|---|
| 案例编号 | 011 |
| 当事人 | 一位大姐 |
| 时间 | 2025 年 |
| 地点 | 某新建大学城 |
| 品类 | 盖浇饭 / 快餐 |
| 投资金额 | 39 万 |
| 最终结局 | 日营业额远低于预期 |

## 起因

大姐看好新建大学城的"未来人流量"，签长租约，开了一家盖浇饭。

预设："学生总要吃饭，日卖 8000 应该没问题。"

## 经过

- 选址在大学城外围，距离生活区有一定距离
- 大学城本身入住率不足
- 学生消费时段集中（午晚高峰共 4 小时）
- 实际日营业额远低于 8000，月度持续亏损

## 关键数据

- 投资：39 万
- 期望日营业额：8000 元
- 实际日营业额：远低于该值
- 房租：长约绑定难退

## 勇哥诊断

- 新开发区 / 新大学城 = 客流要养
- 学生群体客单价低，4 小时高峰能撑起的营业额有上限
- 39 万对于一个尚未跑通的商圈来说太重
- 你应该先去蹲点而不是先签合同

## 失败原因

1. 新开发区客流尚未起量
2. 高估学生群体的复购密度
3. 长租约带来现金流压力
4. 选品同质化高（大学城盖浇饭店通常不止一家）

## 给 skill 的启示

- 用户提到「新大学城 / 新开发区 / 鬼城新盘 / 入住率不足」→ 触发红灯位识别
- 对学生群体客单价默认上限要保守估计
- 长租约 = 风险放大器
