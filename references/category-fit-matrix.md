# 位置类型 × 业态适配矩阵

> 用于回答：这个地段开 X 行不行？不行的话换什么？

先根据图 + 高德把位置归为 **一种主类型**（可并列写次要类型）。

## 位置类型定义

| 代码 | 含义 | 识别线索 |
|------|------|----------|
| school | 学校门口 / 学区外街 | 学校 POI 近、学生流 |
| office | 写字楼 / 办公带 | 写字楼 POI、工作日午高峰 |
| community | 成熟社区底商 | 小区 POI 密、生活配套 |
| food_street | 美食街 / 小吃街 | 餐饮 POI 高密、聚客 |
| mall_upper | 综合体内部 / 二楼+ | 商场 POI、室内/楼层感 |
| traffic_road | 主干道车流型 | 快车道、难停车难停步 |
| new_dev | 新开发区 / 新盘 | 空旷、配套少、在建 |
| monopoly | 封闭客群（封闭校园外唯一餐饮带等） | 替代少、刚需锁死 |

### 使用规则

1. **当前业态红** → 结论至少 🟡，常 🔴；必须给备选。  
2. **备选**：同列里选绿，最多 3 个；优先用户技能门槛低的（早餐/简餐/小吃 > 正餐/概念店）。  
3. **奶茶在 mall_upper 或贴脸大牌** → 默认 🔴（双杀）。  
4. **monopoly + 刚需简餐** → 可 🟢，仍要求确认封闭政策与假期。  
5. **new_dev** → 默认劝缓；除非用户有养铺现金且租极低。  

## 三类生意逻辑（换业态时一句话）

- **小吃** = 流量生意 → 要聚客街/路过密度  
- **刚需快餐** = 复购生意 → 要固定人口（学校/社区/办公）  
- **早餐** = 顺路生意 → 要动线（出门上班上学路过）  

错配例子：流量小吃开进死社区纵深；高客单正餐开学生街；纯夜宵开仅工作日办公区。
