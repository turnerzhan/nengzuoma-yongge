> 从语料库到可运行 skill 的工程化路径。

## B. 三种实现路径

### B.1 Prompt 路径（最轻）

把整个 `corpus/` + `skill/` 拼成一个大 system prompt，直接喂给 GPT-4 / Claude / 通义千问等大模型。

- **优点**：零开发成本，5 分钟可跑
- **缺点**：上下文长、成本高、知识更新需要重发
- **适用**：MVP / Demo / 小范围验证

### B.2 RAG 路径（推荐）

把 `corpus/` 和 `cases/` 作为向量化知识库，`skill/` 作为 prompt 模板：

```
用户输入
 ↓
意图识别（小模型 or LLM）
 ↓
按意图从知识库检索相关段落（3–5 个）
 ↓
填入 prompt 模板（skill/ 里的文件）
 ↓
大模型生成 + 风格约束
 ↓
后处理（算账结果拼接、案例链接附加）
 ↓
输出
```

- **优点**：成本低、知识易更新、可扩展
- **缺点**：需要向量库 + 检索层
- **适用**：正式产品

### B.3 Agent 路径（最强）

在 RAG 基础上增加工具调用能力：

```
工具箱：
- 算账工具（保本线 / 回本期 / 健康度打分）
- 街景识别工具（多模态模型调用）
- 品牌核查工具（企查查 API）
- 案例匹配工具（结构化查询 cases/）
```

- **优点**：最接近真正的"勇哥 AI"
- **缺点**：开发复杂、需要多模态
- **适用**：完整产品 / 商业化

## C. 关键交互设计

### C.1 "360 度转一圈"的实现

```
┌──────────────┐
│ 用户点击触发 │
└──────┬───────┘
 ▼
┌──────────────────────────────────┐
│ 引导：「来，镜头高一点， │
│ 360 度原地转个圈」 │
│ （或：拍 4 张照片：门头/左/右/对街）│
└──────┬───────────────────────────┘
 ▼
┌──────────────────────────────────┐
│ 多模态模型识别 12 项观察点 │
│ （参见 skill/街景观察清单.md） │
└──────┬───────────────────────────┘
 ▼
┌──────────────────────────────────┐
│ 打分 + 匹配最近案例 + 输出勇哥结论 │
└──────────────────────────────────┘
```

**关键视觉识别能力需求**：
- 门头文字识别（OCR）→ 判断是否抽象概念词
- 转让招牌密度（对象检测）
- 台阶数量（几何识别）
- 大品牌 logo 识别（蜜雪 / 瑞幸 / 塔斯汀）
- 街道动线识别（车道 / 人行道宽度）

### C.2 Excel 风格的算账展示

勇哥直播间正上方挂着《这家餐饮店能救吗》的 Excel。skill 应该复现这个视觉锚：

```
┌──────────────────────────────┐
│ 这家餐饮店能救吗？ │
├───────────┬──────────────────┤
│ 日营业额 │ 800 元 [黄] │
│ 日保本线 │ 897 元 │
│ 毛利率 │ 68.75% │
│ 月固定支出 │ 18,500 元 │
│ 保本达成率 │ 89.2% [黄] │
│ 月净利 │ -2,000 元 [红] │
│ 回本周期 │ - [红] │
└───────────┴──────────────────┘
```

## D. 开发优先级建议

### Phase 1（MVP，1 周）
- [ ] 接入大模型（GPT-4 / Claude / 通义千问）
- [ ] 把 `skill/风格指引.md` 做 system prompt
- [ ] 把 `skill/提问树.md` B 节（诊断流程）做成对话状态机
- [ ] `skill/保本线计算器.md` 做成 Python function

### Phase 2（1–2 月）
- [ ] 引入 RAG，向量化 `corpus/` + `cases/`
- [ ] 实现 `skill/提问树.md` 全部 5 条主线
- [ ] 接入企查查 API 做品牌核查
- [ ] 街景文字识别（无视觉模型时降级）

### Phase 3（3–6 月）
- [ ] 多模态视觉识别接入（街景 12 项打分）
- [ ] Excel 风格的前端可视化
- [ ] 用户创业档案持久化（记住每次咨询状态）
- [ ] 案例库众包增长机制

## E. 技术栈建议

### 语言 / 模型
- Python + LangChain / LlamaIndex
- 大模型：Claude 3.5 Sonnet / GPT-4o / 通义千问 Max
- 多模态：GPT-4V / Claude 3.5 / 通义千问 VL
- 嵌入：text-embedding-3 / bge-large-zh

### 向量库
- Chroma（轻量）
- Qdrant / Milvus（生产）

### 前端
- 微信小程序（触达下沉用户最优）
- Web App（PC 咨询场景）
- 飞书 / 钉钉机器人（团队场景）

---

## F. 风险与红线

### F.1 法律风险
- 不构成专业投资建议 / 法律建议
- 不替代专业律师 / 会计师
- 涉及具体品牌定性时必须基于公开报道

### F.2 伦理风险
- 用户情绪极端（如"我就想死"）时必须转向心理援助
- 不嘲讽受害者，只批评行为
- 不泄露当事人隐私

### F.3 内容风险
- 案例库的品牌名涉及商誉，必须标注"公开报道"出处
- 不主动指控未定性品牌，只列"风险信号"
- 任何新品牌判断必须**多源交叉**

---

## G. 成功指标

- **短期**：用户完成诊断的比例 > 70%
- **中期**：用户真实听劝比例（上线后追踪）
- **长期**：**避免损失金额 / 每次咨询**（这是真正的产品价值）
