# 贡献指南

> 欢迎一起把这个语料库越养越肥。

## 你可以贡献什么

### 1. 新的案例（最受欢迎）

如果你在勇哥直播切片、抖音、B 站、知乎看到了一个**有信息量**的案例，欢迎提 PR 添加。

模板见 [`cases/失败案例/_模板.md`](./cases/失败案例/_模板.md)（如果还没有，参考已有案例的结构）。

**好案例的标准**：
- 有明确金额（投入了多少、亏了多少）
- 有品类、地点、时间
- 有失败/成功的具体原因
- 有勇哥的原话/判断节选
- 有可溯源的链接（B 站切片、抖音视频、文章 URL）
- 别只写"有个大姐开店亏了"——这没价值

### 2. 新的金句

补到 [`corpus/06-金句与话术.md`](./corpus/06-金句与话术.md)。**必须带出处**，最好附场景描述。

格式：
```markdown
> 金句正文

- 出处：B 站切片 / 抖音 / 直播 X 月 X 日
- 场景：勇哥在点评 XXX 案例时说的
- 链接：（如有）
```

### 3. 街景观察清单的补充

[`skill/街景观察清单.md`](./skill/街景观察清单.md) 里的 12 个观察点是初版。如果你发现勇哥经常提但清单里没列的点（比如"看店里有没有装监控"这种），欢迎补充。

每个新观察点必须包含：
- 观察项名称
- 红/黄/绿灯阈值
- 勇哥原话锚点（出处）
- 至少 1 个反例案例

### 4. 方法论的修正

如果你发现 [`corpus/`](./corpus/) 里某个方法论描述有误（数字算错、引用失真、阶段拆解错），直接提 PR。**改方法论必须附原始视频/文章链接**。

### 5. Skill 实现层的样例

如果你已经基于这个语料库做出了实际的 Agent / Skill / Bot，欢迎在 [`docs/`](./docs/) 下提交一个使用案例。

## 怎么提

1. Fork 仓库
2. 新建分支：`git checkout -b feat/case-XXX` 或 `feat/quote-XXX`
3. 按照模板填写
4. 提 PR，标题前缀：
- `[case]` 新增案例
- `[quote]` 新增金句
- `[sop]` 方法论变更
- `[skill]` skill 资源
- `[fix]` 修错
- `[docs]` 文档
5. 等 review

## 写作风格

- 用**数字**说话，能写"日营业额 800"就别写"很差"
- 用**短句**，参考勇哥本人的语言节奏
- **不洗白、不美化、不留情面**——这是这个项目的气质
- 但也**不嘲讽真实困难**——勇哥骂的是骗子和不听劝的人，不是穷人

## 行为准则

- 不发布人肉/骚扰/恶意攻击当事人的内容
- 案例里的当事人尽量用化名或网友昵称
- 不传播未经证实的信息，所有指控类内容必须有公开来源
- 商业品牌相关内容必须基于公开报道

## 致谢

每一个贡献者都会被列在 [README](./README.md) 的致谢区。

> 让更多想创业的普通人，**在掏钱之前**先看到这个仓库——这就是你能给这个世界的一份善意。
