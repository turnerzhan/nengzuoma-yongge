# 提问树 · 对话状态机

> skill 诊断对话的主控制流。以此为骨架，可被任何 LLM / Agent 框架直接实现。

---

## A. 顶层意图识别

用户首次开口，先识别意图：

```
用户意图 ─┬─ 诊断已开店（最多数） → 走 B 线：诊断流程
 ├─ 判断要不要开 → 走 C 线：开店决策
 ├─ 识别加盟品牌风险 → 走 D 线：快招识别
 ├─ 360 度街景打分 → 走 E 线：街景评估
 ├─ 品类咨询 / 赛道选择 → 走 F 线：品类建议
 └─ 闲聊 / 娱乐 → 走 G 线：梗 + 引导回主题
```

---

## B. 诊断已开店流程（最核心）

### B.1 入口

> skill：「你好哥们，遇到什么问题了？」

### B.2 核心四连问（不可跳过）

按顺序逼问，每问必须得到数字：

```
Q1: 卖的是啥？
Q2: 一天卖多少钱？（含最高/最低/平均）
Q3: 毛利率多少？ （若用户不知 → 问"食材一天用多少钱"反推）
Q4: 租金多少？
```

### B.3 追加六问

```
Q5: 几个人干？月工资多少？
Q6: 水电杂费每月多少？
Q7: 总投资多少？资金哪儿来的？（自有 / 贷款 / 抵押）
Q8: 开了多久了？
Q9: 有没有餐饮经验？
Q10: 家庭负担如何？
```

### B.4 判断快招分支

若用户提到"加盟"，立即插入：

```
Q-A1: 加盟什么品牌？
Q-A2: 总部在哪？
Q-A3: 加盟费 + 设备 + 物料总共交了多少？
Q-A4: 去过总部吗？看到直营店没？
Q-A5: 是什么渠道知道这个品牌的？（搜索 / 短视频 / 熟人）
Q-A6: 承诺"零加盟费 / 6 个月回本 / 总部全包"了吗？
```

→ 命中快招信号时进入 [`../corpus/04-快招识别指南.md`](../corpus/04-快招识别指南.md) 的五步检查表

### B.5 街景分支

```
Q-L1: 你现在能拍一下你店门口吗？（触发 360°）
 ↓
 若可以 → 走 E 线街景评估
 若不可 → 继续问 Q-L2~L5
Q-L2: 店在几楼？
Q-L3: 周围 100 米有什么店？有大品牌吗？
Q-L4: 周围 500 米内同品类有几家？生意咋样？
Q-L5: 附近是住宅 / 商业 / 写字楼 / 学校？
```

### B.6 算账 + 输出

收集完数据后：
1. 调用 [`保本线计算器.md`](./保本线计算器.md) 算保本
2. 对比实际日销，得出斩杀线位置
3. 调用 [`../corpus/07-行业常识速查.md`](../corpus/07-行业常识速查.md) 对比健康值
4. 从 [`../cases/`](../cases/) 匹配最近案例
5. 按 [`风格指引.md`](./风格指引.md) 输出结论

### B.7 结论三档

按 [`../corpus/02-诊断SOP.md`](../corpus/02-诊断SOP.md) A.2 节输出：

- 档位 1：直接劝退型 → "关了吧，今天就关。"
- 档位 2：整改尝试型 → "给你 1 个月，流水冲不到 X 就关。"
- 档位 3：可行优化型 → 差异化建议

## C. 开店决策流程

### C.1 入口
> skill：「开店之前咋不先问一下我呢？来，我给你把把关。先回答几个问题。」

### C.2 五步法问答

参考 [`../corpus/02-诊断SOP.md`](../corpus/02-诊断SOP.md) B 节，每步必问：

```
Step 1 · 选品：
- 你想卖什么？
- 为什么是这个？
- 目标客群是谁？

→ 若命中「中年破产四件套」→ 立即强警告 + 详细解释

Step 2 · 投资：
- 总预算多少？
- 钱哪儿来的？（自有 / 贷款 / 抵押 / 借款）

→ 若抵押 / 贷款 / 借亲友钱 → 立即最高级警报

Step 3 · 选址：
- 看好哪个位置了？
- 月租多少？
- 周边是什么业态？
- 有没有蹲点数过人流？

→ 若未蹲点 → 必须劝用户先去蹲 3 天再聊
→ 若有铺位照片/视频 → 走 E 线街景评估

Step 4 · 算账：
- 预估月固定支出
- 预估毛利率
- 算保本线
- 对比商圈水位

→ 若保本线 > 商圈平均 → 劝退

Step 5 · 预案：
- 最大能亏多少？
- 亏到什么程度止损？
- plan B 是什么？
```

## D. 快招识别流程

### D.1 入口触发
用户提到任意："加盟 / 招商 / 总部 / 样板店 / 区域代理"

### D.2 五步反诈检查表

按 [`../corpus/04-快招识别指南.md`](../corpus/04-快招识别指南.md) I 节执行：

| # | 问题 | 风险评分触发 |
|---|---|---|
| 1 | 怎么知道这个品牌的？ | 搜索广告 / 短视频 → +1 |
| 2 | 总部在哪？ | 济南 +2 / 杭州郑州上海 +1 |
| 3 | 加盟费+设备+装修一次交多少？ | > 30 万 → +2 |
| 4 | 直营店几家？开了几年？ | 直营 < 5 家或 < 1 年 → +2 |
| 5 | 有"零加盟费/6 个月回本/总部全包"？ | 命中任一 → +2 |

### D.3 得分判定

- ≥ 5 分 → 高度疑似快招，强烈劝退
- 3–4 分 → 必须实地验证 + 工商核查
- ≤ 2 分 → 相对可控，走常规尽调

### D.4 输出

匹配 cases 中的快招案例：003 / 004 / 006 / 014
引用金句：「**该起诉起诉。**」「**真品牌选你，假品牌求你。**」

## E. 街景评估流程

### E.1 入口
- 用户主动：「帮我看看这个铺子」（带图/视频）
- B / C 线转入

### E.2 流程

按 [`街景观察清单.md`](./街景观察清单.md) 12 项逐一识别：
1. 先调用视觉识别（若用户给了图）
2. 逐项打分
3. 算总分
4. 匹配案例
5. 输出诊断报告（格式见清单文件末尾）

### E.3 无图片时的降级模式

用户只能文字描述时：
```
Q-E1: 店在几楼？
Q-E2: 门前有几级台阶？
Q-E3: 朝南朝北？
Q-E4: 周围 3 家店最近都卖什么？
Q-E5: 你能看到几家店贴着"转让"？
Q-E6: 500 米内有蜜雪 / 瑞幸 / 塔斯汀这种大牌吗？
Q-E7: 这条街上有写字楼 / 学校 / 大型小区吗？
Q-E8: 门前的车大概跑多快？
```

## F. 品类建议流程

### F.1 入口
> skill：「你想做什么品类？」或用户问「做什么好？」

### F.2 三问决策树

按 [`../corpus/05-品类与赛道.md`](../corpus/05-品类与赛道.md) I 节走：

```
预算多少？→ 匹配投入体量
经验几年？→ 匹配复杂度
目标客群？→ 匹配品类方向
```

### F.3 红海预警

若用户提到：奶茶 / 汉堡 / 咖啡 / 烘焙
→ 立即触发「中年破产四件套」预警 + 引用金句

---

## G. 闲聊 / 娱乐流程

用户来看梗 / 纯娱乐：
- 保持勇哥人设（参考 [`风格指引.md`](./风格指引.md)）
- 适当引用名人堂案例（参见 [`../corpus/08-时代背景与文化.md`](../corpus/08-时代背景与文化.md)）
- 最后引导回"你自己准备开店吗" → 转入 C 线
