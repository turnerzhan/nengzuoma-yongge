# 街景观察清单 · 12 项打分模型

> skill 核心交互「**360 度转一圈**」的直接依据。用户上传街景照片/视频，skill 按此清单逐项打分。

---

## 使用方式

用户操作：
1. 站在目标铺位正门前
2. 对着街道 360° 旋转拍一圈视频（或拍几张照片：门头正面 / 左右两侧 / 对街）
3. 发送给 skill

skill 处理：
- 按下方 12 项逐一识别 + 打分
- 输出红 / 黄 / 绿灯 + 总分
- 匹配案例 + 给出勇哥式结论

---

## 12 项观察清单

### 01 · 门头宽度

| 阈值 | 颜色 | 判定 |
|---|---|---|
| ≥ 1.5 米 | [绿] 绿灯 | 正常，视觉充足 |
| 1.2–1.5 米 | [黄] 黄灯 | 勉强及格，小吃可以 |
| < 1.2 米 | [红] 红灯 | 太窄，进店率和识别率都降低 |

> 勇哥原话：**门头要宽店要窄。**

反例：cases 中多个"棺材铺"（一楼窄二楼大）案例

---

### 02 · 招牌信息层级

| 判断 | 颜色 | 说明 |
|---|---|---|
| 品类字最大 + 品牌次之 + 亮度足够 | [绿] | 路人 1 秒知道卖什么 |
| 品牌字大 + 品类字小 | [黄] | 损失部分路人 |
| 主打"概念文字"（哲学 / 文化 / 长句） | [红] | 典型"家是本"陷阱 |

> 勇哥原话（对家是本）：「**你这门头满是文字，'巨大历史机遇'比'葱油饼'还大。**」

反例：[cases/失败案例/005-家是本-成都葱油饼.md](../cases/失败案例/005-家是本-成都葱油饼.md)

---

### 03 · 台阶数

| 台阶数 | 颜色 | 判定 |
|---|---|---|
| 0–1 级 | [绿] | 无阻碍 |
| 2 级 | [黄] | 小损失 |
| ≥ 3 级 | [红] | **每级约减少 30% 进店率** |

> 地面平齐 > 一级小台阶 > 多级台阶 > 半地下 > 二楼

---

### 04 · 门前遮挡物

识别：变压器 / 电线杆 / 绿化带 / 栏杆 / 公交站 / 垃圾桶

| 遮挡情况 | 颜色 |
|---|---|
| 无遮挡 | [绿] |
| 1 个小遮挡（距门 > 3 米） | [黄] |
| 多个遮挡 / 紧贴门口 | [红] |

> 勇哥原话：「**门前有栏杆的铺子慎租**——人过来要绕 10 米。」

---

### 05 · 阴街 / 阳街判定

观察影子 + 朝向：

| 判断 | 颜色 |
|---|---|
| 朝南 / 朝东，白天有阳光 = 阳街 | [绿] |
| 朝西，下午晒（夏天吃亏） | [黄] |
| 朝北 / 被高楼挡全天阴冷 = 阴街 | [红] |

> 同一条街两侧租金可能差 30–50%，**阳街值得多花钱**。

---

### 06 · 转让招牌密度（死街检测）

目视周围可见店铺，统计贴"转让"的数量：

```
转让率 = 贴转让店数 / 可见店总数
```

| 转让率 | 颜色 | 判定 |
|---|---|---|
| < 15% | [绿] | 商圈活 |
| 15%–30% | [黄] | 商圈疲软 |
| > 33% | [红] | **死街预警**，整条街客流已塌 |

---

### 07 · 同业浓度

周围 500 米内同品类店铺数量：

| 数量 | 颜色 | 判定 |
|---|---|---|
| < 10 | [红] | 商圈不成熟，先驱者变先烈 |
| 10–40 | [黄] | 起步期，谨慎 |
| **40–120** | [绿] | **黄金区间** |
| > 120 | [红] | 内卷红海 |

---

### 08 · 销售推动业态

500 米内是否有：写字楼 / 学校 / 大型小区（1500 户+）/ 商场主力店

| 叠加数 | 颜色 |
|---|---|
| ≥ 2 个 | [绿] 客流叠加 |
| 1 个 | [黄] 单点依赖 |
| 0 个 | [红] 无客流来源 |

---

### 09 · 竞品识别（大品牌在场？）

识别附近 50 米内是否有：**蜜雪冰城 / 瑞幸 / 塔斯汀 / 华莱士 / 霸王茶姬 / 古茗**

| 情况 | 颜色 | 说明 |
|---|---|---|
| 你做的品类没有大牌竞品 | [绿] | 竞争缓和 |
| 有同品类大牌但你走差异化 | [黄] | 可做，但得说清楚差异化是什么 |
| **你做的是"杂牌 + 同品类 + 紧挨大牌"** | [红] | **自杀式选址**（参见脸盆姐） |

---

### 10 · 楼层与动线

| 情况 | 颜色 |
|---|---|
| 一楼临街，正门朝主干道 | [绿] |
| 一楼但在巷子里 / 侧门 | [黄] |
| 二楼（餐饮）/ 楼梯口 / 死角 | [红] |
| **整栋楼 / 3 层以上（杂牌）** | [红红] 典型七层奶茶大厦 |

---

### 11 · 证照与基础设施

需在店内 / 店旁确认：
- 营业执照
- 食品经营许可证 （双证缺一不租）
- 排烟烟道是否合规
- 排水是否有隔油池 / 能否接通
- 用电是否商业电 / 三相电（大功率设备必需）

| 情况 | 颜色 |
|---|---|
| 双证齐 + 三设施齐 | [绿] |
| 双证齐但设施需改造 | [黄] |
| **缺证 / 缺关键设施** | [红] 绝对不租 |

---

### 12 · 门前路况

| 路况 | 颜色 |
|---|---|
| 人行道 + 慢车道（车速 < 30 码）+ 有停车位 | [绿] |
| 中等车道（30–40 码）+ 有临停 | [黄] |
| 快车道（车速 50+ 码） / 无人行道 | [红] 看得见进不来 |

---

## 综合打分算法

```
单项打分：
[绿] = 10 分
[黄] = 5 分
[红] = 0 分

总分 = Σ 12 项 × 权重

权重建议：
核心项（×1.5）：05 阴阳街、06 转让密度、07 同业浓度、09 竞品、10 楼层
标准项（×1.0）：01 门头、02 招牌、03 台阶、04 遮挡、08 推动业态、12 路况
硬性项（×2.0）：11 证照（一票否决：缺证直接判 0 分整体）
```

### 总分分档

| 总分 | 判定 | 勇哥诊断模板 |
|---|---|---|
| 80–100 | [绿] 好铺 | 「**没问题。**」甚至「**WC，关起来杀啊！**」（如命中封闭客流） |
| 60–79 | [黄] 可尝试 | 「勉强能做，但你得有差异化策略。」 |
| 40–59 | [橙] 高风险 | 「**做不了**，除非租金砍到一半。」 |
| 20–39 | [红] 劝退 | 「**关了吧，今天就关。**」（如果已签约就止损） |
| < 20 | [黑] 韭菜斩杀线 | 「你这店从你开的那一刻就已经死了。」 |

## 输出模板（给 skill）

```
---
 360° 街景诊断报告
---
[绿] 门头宽度：~1.8m → 充足（10 分）
[红] 台阶：3 级 → 每级损失 30% 进店（0 分 × 1.0）
[黄] 阴阳街：朝西下午晒（5 分 × 1.5 = 7.5）
[红] 转让密度：4/12 = 33% 死街预警（0 分 × 1.5）
[绿] 同业浓度：58 家（黄金区）（10 分 × 1.5）
[黄] 竞品：50m 内蜜雪（5 分 × 1.5 = 7.5）
[绿] 楼层：一楼临街（10 分 × 1.5）
[绿] 证照：双证齐全
...

---
总分：58 / 100 → [橙] 高风险
---
 勇哥诊断：
做不了哥们。这个位置最大的问题是——
整条街 1/3 都在转让，这叫死街。
你开进去就是给房东送钱。
除非你拿到的是真实月租的 6 折以下。

 最像的案例：[002 七层奶茶大厦](../cases/失败案例/002-七层奶茶大厦-长治.md)
```

## 适用 skill 场景

- 用户上传街景视频/照片 → 逐项识别 + 打分
- 用户描述周边环境（无图） → 按 12 项逐个问 → 打分
- 输出时保持视觉化（红黄绿表情符 + 分数 + 诊断语）
